Foundations

1. What is an Observable?

An Observable is a lazy data stream that can emit zero or more values over time, either synchronously or asynchronously. Think of it as a function that can return multiple values instead of just one.

💡

Key mental model: A Promise gives you one value, once, and starts immediately. An Observable gives you many values, over time, and starts only when you subscribe.

Creating an Observable from scratch

basic-observable.ts 
import { Observable } from 'rxjs';

// The function inside new Observable() is called the "subscriber function"
// It runs fresh for EVERY new subscription (lazy)
const numbers$ = new Observable<number>(subscriber => {
  subscriber.next(10);
  subscriber.next(20);
  subscriber.next(30);
  subscriber.complete(); // stream ends
});

// Nothing runs until you subscribe
numbers$.subscribe(value => console.log(value));
// Output: 10, 20, 30

Observable vs Promise — side by side

vs-promise.ts 
// ── PROMISE ──────────────────────────────────────────────
const promise = new Promise(resolve => {
  console.log('Promise executes NOW');  // runs immediately
  resolve(42);
});
// "Promise executes NOW" already printed

promise.then(v => console.log(v)); // 42

// ── OBSERVABLE ───────────────────────────────────────────
const observable$ = new Observable(subscriber => {
  console.log('Observable executes LAZILY'); // not yet
  subscriber.next(42);
});
// Nothing printed yet

observable$.subscribe(v => console.log(v));
// NOW "Observable executes LAZILY" prints, then 42
Feature Observable Promise
Values emitted 0 to many Exactly 1
Execution Lazy (on subscribe) Eager (immediately)
Cancellable Yes (unsubscribe) No
Synchronous Yes No (always async)
Operators Rich (map, filter…) Limited (.then, .catch)

2. Observer & Subscription

An Observer is the consumer — an object with three optional callbacks. A Subscription is the handle returned when you subscribe, which lets you cancel.

Full observer object

observer.ts 
import { Observable, Observer } from 'rxjs';

const observer: Observer<string> = {
  next:     (value) => console.log('Value:', value),
  error:    (err)   => console.error('Error:', err),
  complete: ()      => console.log('Stream done!'),
};

const data$ = new Observable<string>(sub => {
  sub.next('Hello');
  sub.next('World');
  sub.complete();
});

data$.subscribe(observer);
// Value: Hello  |  Value: World  |  Stream done!

Using the Subscription to unsubscribe

subscription.ts 
import { interval, Subscription } from 'rxjs';

// interval emits 0, 1, 2, 3... every 1 second — never completes
const sub: Subscription = interval(1000).subscribe(n =>
  console.log('tick:', n)
);

// Stop after 5 seconds
setTimeout(() => {
  sub.unsubscribe();
  console.log('Unsubscribed — no more ticks');
}, 5000);

Teardown logic — cleanup when unsubscribed

teardown.ts 
const websocket$ = new Observable(subscriber => {
  const ws = new WebSocket('ws://localhost:8080');
  ws.onmessage = (e) => subscriber.next(e.data);
  ws.onerror  = (e) => subscriber.error(e);

  // Return function runs on unsubscribe / complete / error
  return () => {
    console.log('Closing WebSocket');
    ws.close();
  };
});

const sub = websocket$.subscribe(msg => console.log(msg));
sub.unsubscribe(); // → "Closing WebSocket"

3. Cold vs Hot Observables

Type When does it start? Is data shared? Examples
Cold Each subscriber gets its own execution No — each sub gets fresh values HTTP calls, of(), from()
Hot Already running, subscribers tap in Yes — all subs share the same stream Subject, fromEvent(), WebSocket

Cold: each subscriber gets its own copy

cold.ts 
import { Observable } from 'rxjs';

const cold$ = new Observable(sub => {
  sub.next(Math.random()); // different number each time
  sub.complete();
});

cold$.subscribe(v => console.log('Sub A:', v)); // 0.423
cold$.subscribe(v => console.log('Sub B:', v)); // 0.819 (different!)

// HttpClient works the same — two subscribers = two HTTP requests

Hot: all subscribers share the same stream

hot.ts 
import { Subject } from 'rxjs';

const hot$ = new Subject<number>();

hot$.subscribe(v => console.log('Sub A:', v));
hot$.subscribe(v => console.log('Sub B:', v));

hot$.next(99);
// Sub A: 99
// Sub B: 99  ← both got the same value from the same event
Creation Operators

4. of / from / fromEvent

of — emit a fixed list of values

of.ts 
import { of } from 'rxjs';

// Synchronously emits each argument then completes
of(1, 2, 3).subscribe(console.log); // 1, 2, 3

of('Loading...').subscribe(msg => (this.status = msg));

// Angular: useful for fallback values in streams
const user$ = this.userService.getUser().pipe(
  catchError(() => of(null)) // fallback to null on error
);

from — convert arrays, Promises, or iterables

from.ts 
import { from } from 'rxjs';

// From array: emits each element
from(['Alice', 'Bob', 'Carol'])
  .subscribe(name => console.log(name));
// Alice, Bob, Carol

// From Promise: wraps a promise as an observable
from(fetch('/api/users').then(r => r.json()))
  .subscribe(users => console.log(users));

// From string: emits each character
from('hello').subscribe(console.log);
// h, e, l, l, o

fromEvent — DOM events as streams

fromEvent.ts 
import { fromEvent } from 'rxjs';
import { debounceTime, map } from 'rxjs/operators';

// Every keypress on the search input as an Observable
const input = document.getElementById('search')!;

fromEvent(input, 'keyup').pipe(
  debounceTime(300),
  map((e) => (e.target as HTMLInputElement).value)
).subscribe(query => this.search(query));

// In Angular, use reactive forms instead for input — fromEvent
// is most useful for window/document-level events:
fromEvent(window, 'resize').pipe(
  debounceTime(100),
  map(() => window.innerWidth)
).subscribe(width => this.screenWidth = width);

5. interval / timer / EMPTY

interval — ticks every N milliseconds

interval.ts 
import { interval } from 'rxjs';
import { take, map } from 'rxjs/operators';

// Polls an API every 10 seconds — common Angular pattern
interval(10000).pipe(
  switchMap(() => this.http.get<Status>('/api/status'))
).subscribe(status => this.status = status);

// Countdown timer
interval(1000).pipe(
  take(10),
  map((i) => 10 - i)
).subscribe(n => this.countdown = n);

timer — delay then optionally repeat

timer.ts 
import { timer } from 'rxjs';

// Fires once after 3 seconds
timer(3000).subscribe(() => this.showBanner = false);

// First emission after 2s, then every 5s (like interval but delayed)
timer(2000, 5000).pipe(
  switchMap(() => this.http.get('/api/notifications'))
).subscribe(...);

EMPTY — completes immediately, emits nothing

empty.ts 
import { EMPTY } from 'rxjs';

// Guard a stream — do nothing if condition not met
this.form.valueChanges.pipe(
  switchMap((val) => {
    if (!val.query.trim()) return EMPTY; // skip empty queries
    return this.searchService.search(val.query);
  })
).subscribe(results => this.results = results);

6. Subjects

A Subject is both an Observable and an Observer. You can push values into it manually, and multiple subscribers share the same stream (hot).

Type Replays to new subscribers? Initial value? Common use
Subject No — only future values No Events, user actions
BehaviorSubject Yes — last 1 value Required Current state (most common)
ReplaySubject(n) Yes — last n values No Caching recent history
AsyncSubject Yes — last value on complete No Rare: one-shot result

BehaviorSubject — the most useful one

behavior-subject.ts 
import { BehaviorSubject } from 'rxjs';

// Must have an initial value
const count$ = new BehaviorSubject<number>(0);

// Subscribe A before any values are pushed
count$.subscribe(v => console.log('A:', v)); // A: 0 (immediately!)

count$.next(1); // A: 1
count$.next(2); // A: 2

// Late subscriber gets the LAST value immediately
count$.subscribe(v => console.log('B:', v)); // B: 2

// Read current value synchronously (no subscribe needed)
console.log(count$.value); // 2

State service pattern with BehaviorSubject

cart.service.ts 
@Injectable({ providedIn: 'root' })
export class CartService {
  // Private: only this service can push new values
  private items$ = new BehaviorSubject<CartItem[]>([]);

  // Public: components only read, never write directly
  readonly items = this.items$.asObservable();
  readonly count$ = this.items.pipe(map((i) => i.length));

  addItem(item: CartItem) {
    this.items$.next([...this.items$.value, item]);
  }

  clear() {
    this.items$.next([]);
  }
}

// In component template:
// <span>{{ cartService.count$ | async }} items</span>

ReplaySubject — buffer recent history

replay-subject.ts 
import { ReplaySubject } from 'rxjs';

// Keeps last 3 values in buffer
const log$ = new ReplaySubject<string>(3);

log$.next('Event 1');
log$.next('Event 2');
log$.next('Event 3');
log$.next('Event 4'); // 1 is dropped, buffer is [2, 3, 4]

// Late subscriber gets the buffered 3 immediately
log$.subscribe(e => console.log(e));
// Event 2, Event 3, Event 4
Operators

7. map / filter / tap

These are the bread-and-butter operators. They live inside .pipe() and transform every emission.

map — transform each value

map.ts 
import { map } from 'rxjs/operators';

// Transform HTTP response to just the data you need
this.http.get<ApiResponse>('/api/employees').pipe(
  map((res) => res.data),                          // unwrap wrapper
  map((employees) => employees.filter(e => e.active)) // filter array
).subscribe(employees => this.employees = employees);

// Map router params to a number
this.route.paramMap.pipe(
  map((params) => Number(params.get('id')))
).subscribe(id => this.loadEmployee(id));

filter — only let matching values through

filter.ts 
import { filter } from 'rxjs/operators';

// Only react to keyboard shortcuts
fromEvent<KeyboardEvent>(document, 'keydown').pipe(
  filter((e) => e.ctrlKey && e.key === 's')
).subscribe(() => this.save());

// Only emit non-null values (type-safe null check)
this.userService.currentUser$.pipe(
  filter((user): user is User => user !== null)
).subscribe((user) => this.loadProfile(user.id));

tap — side effects without changing the stream

tap.ts 
import { tap } from 'rxjs/operators';

// Logging / debugging — tap doesn't alter the value
this.http.get<User[]>('/api/users').pipe(
  tap((users) => console.log('Users loaded:', users.length)),
  tap(() => this.loading = false),         // hide spinner
  map((users) => users.filter(u => u.active))
).subscribe((users) => this.users = users);

8. Higher-order mapping operators

These operators map each emission to a new Observable, then flatten the result. The difference is in how they handle overlapping inner observables.

⚠️

This is the most important group to master. Choosing the wrong one causes stale responses, missed clicks, or doubled requests.

Operator Overlap strategy Best for
switchMap Cancels previous inner observable Search, autocomplete, nav
mergeMap Runs all concurrently Parallel independent requests
concatMap Queues — waits for each to finish Ordered sequential actions
exhaustMap Ignores new until current finishes Login, form submit, prevent double-click

switchMap — cancel the previous, use the latest

switchMap.ts 
import { switchMap, debounceTime } from 'rxjs/operators';

// Classic search: if user types again before the HTTP response,
// the previous request is CANCELLED — no stale results
this.searchControl.valueChanges.pipe(
  debounceTime(300),
  switchMap((query) => this.http.get<Result[]>(`/api/search?q=${query}`))
).subscribe((results) => this.results = results);

// Also used for route-driven data loading
this.route.paramMap.pipe(
  switchMap((params) =>
    this.empService.getEmployee(Number(params.get('id')))
  )
).subscribe((emp) => this.employee = emp);

mergeMap — run all concurrently

mergeMap.ts 
import { mergeMap, from } from 'rxjs';

// Upload multiple files simultaneously — all requests fire at once
from(this.selectedFiles).pipe(
  mergeMap((file) => this.uploadService.upload(file))
).subscribe({
  next:     (result) => this.uploaded.push(result),
  complete: ()       => this.showSuccess('All files uploaded')
});

concatMap — queue one at a time, in order

concatMap.ts 
import { concatMap, from } from 'rxjs';

// Save steps in strict order: step 2 only starts after step 1 finishes
from([step1Payload, step2Payload, step3Payload]).pipe(
  concatMap((payload) => this.http.post('/api/save', payload))
).subscribe({
  next:     (res) => console.log('Step saved:', res),
  complete: ()    => console.log('All steps done in order')
});

exhaustMap — ignore while busy

exhaustMap.ts 
import { exhaustMap, fromEvent } from 'rxjs';

// Prevent double-submit: if login is in progress, extra clicks are ignored
fromEvent(this.loginBtn.nativeElement, 'click').pipe(
  exhaustMap(() => this.authService.login(this.credentials))
).subscribe({
  next:  (res) => this.router.navigate(['/home']),
  error: (err) => this.showError(err)
});

9. Time operators

debounceTime — wait for silence

debounce.ts 
import { debounceTime, distinctUntilChanged } from 'rxjs/operators';

// Only emit after user stops typing for 400ms
this.searchControl.valueChanges.pipe(
  debounceTime(400),
  distinctUntilChanged() // skip if value didn't change
).subscribe((query) => this.doSearch(query));
// User types "ang" — only fires once after 400ms, not on each key

throttleTime — at most once per interval

throttle.ts 
import { throttleTime } from 'rxjs/operators';

// Handle scroll events at most once every 100ms
fromEvent(window, 'scroll').pipe(
  throttleTime(100)
).subscribe(() => this.checkScrollPosition());

// debounce vs throttle:
// debounce: waits for user to STOP typing → good for search
// throttle: fires at a regular pace while events continue → good for scroll/resize

distinctUntilChanged — skip duplicate consecutive values

distinct.ts 
import { distinctUntilChanged } from 'rxjs/operators';

// Avoid re-rendering if the user ID didn't change
this.authService.user$.pipe(
  map((u) => u?.id),
  distinctUntilChanged() // only emits if id changed
).subscribe((id) => this.loadUserData(id));

// With a custom comparator
formControl.valueChanges.pipe(
  distinctUntilChanged((a, b) => a.toLowerCase() === b.toLowerCase())
).subscribe(...);

10. Limiting operators

take — complete after N values

take.ts 
import { take, first } from 'rxjs/operators';

// Take only the first emission, then auto-complete
this.userService.user$.pipe(
  take(1) // equivalent to first()
).subscribe((user) => this.doOneTimeInit(user));

// first() — same as take(1) but can include a predicate
this.items$.pipe(
  first((items) => items.length > 0) // first non-empty array
).subscribe(...);

takeUntil — complete when a notifier emits

takeUntil.ts 
import { Subject } from 'rxjs';
import { takeUntil } from 'rxjs/operators';

@Component({ ... })
export class MyComponent implements OnDestroy {
  private destroy$ = new Subject<void>();

  ngOnInit() {
    // Auto-unsubscribes when destroy$ emits
    interval(1000).pipe(
      takeUntil(this.destroy$)
    ).subscribe((n) => console.log(n));
  }

  ngOnDestroy() {
    this.destroy$.next();    // triggers completion
    this.destroy$.complete(); // cleanup Subject itself
  }
}

takeUntilDestroyed — Angular 16+ (recommended)

takeUntilDestroyed.ts 
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
import { inject, DestroyRef } from '@angular/core';

@Component({ ... })
export class MyComponent {
  private destroyRef = inject(DestroyRef); // no OnDestroy needed

  ngOnInit() {
    interval(1000).pipe(
      takeUntilDestroyed(this.destroyRef)
    ).subscribe((n) => console.log(n));
    // Automatically unsubscribes when component is destroyed
  }
}

11. Combination operators

combineLatest — combine latest from multiple streams

combineLatest.ts 
import { combineLatest } from 'rxjs';

// Emits whenever EITHER stream updates — waits for all to emit once first
combineLatest({
  user:    this.userService.currentUser$,
  filters: this.filterService.activeFilters$,
  page:    this.paginationService.page$,
}).pipe(
  switchMap(({ user, filters, page }) =>
    this.dataService.getData({ userId: user.id, filters, page })
  )
).subscribe((data) => this.tableData = data);

forkJoin — parallel requests, wait for all to finish

forkJoin.ts 
import { forkJoin } from 'rxjs';

// Load all data needed for a page in parallel
// Emits once when ALL complete, with the last value of each
forkJoin({
  employee:    this.http.get<Employee>(`/api/employees/${id}`),
  departments: this.http.get<Department[]>('/api/departments'),
  roles:        this.http.get<Role[]>('/api/roles'),
}).subscribe(({ employee, departments, roles }) => {
  this.employee    = employee;
  this.departments = departments;
  this.roles        = roles;
  this.loading      = false;
});

// IMPORTANT: forkJoin requires all streams to COMPLETE.
// HttpClient does — but Subject/BehaviorSubject never does.

withLatestFrom — combine on one stream's timing

withLatestFrom.ts 
import { withLatestFrom } from 'rxjs/operators';

// When save button is clicked, grab the latest form value + user
this.saveClicks$.pipe(
  withLatestFrom(this.form.valueChanges, this.userService.user$)
).subscribe(([_click, formValue, user]) => {
  this.http.post('/api/save', { ...formValue, userId: user.id });
});

12. Error handling

💡

When an Observable errors, the stream terminates. Error handling operators let you recover and keep the stream alive.

catchError — recover and continue

catchError.ts 
import { catchError } from 'rxjs/operators';
import { of, EMPTY } from 'rxjs';

// Return empty array on error — page shows nothing instead of crashing
this.http.get<User[]>('/api/users').pipe(
  catchError((err) => {
    console.error(err);
    return of([]); // fallback value
  })
).subscribe((users) => this.users = users);

// Rethrow after doing something (e.g. logging)
this.http.get('/api/data').pipe(
  catchError((err) => {
    this.logService.log(err);
    this.errorMsg = err.message;
    return EMPTY; // complete silently without a value
  })
);

retry — automatically retry on error

retry.ts 
import { retry, catchError } from 'rxjs/operators';
import { of } from 'rxjs';

// Retry up to 3 times before giving up
this.http.get<Data>('/api/flaky-endpoint').pipe(
  retry(3),
  catchError((err) => {
    this.errorMsg = 'Failed after 3 attempts';
    return of(null);
  })
).subscribe(...);

finalize — always runs (like try/finally)

finalize.ts 
import { finalize } from 'rxjs/operators';

// Hide spinner whether request succeeds, fails, or is cancelled
this.loading = true;
this.http.get<User[]>('/api/users').pipe(
  finalize(() => this.loading = false) // always runs at the end
).subscribe({
  next:  (users) => this.users = users,
  error: (err)   => this.errorMsg = err.message,
});

13. shareReplay — multicast & cache

shareReplay makes a cold observable hot and caches the last N emissions for new subscribers. Critical for avoiding duplicate HTTP requests.

shareReplay.ts 
import { shareReplay } from 'rxjs/operators';

// Without shareReplay: 3 subscribers = 3 HTTP requests
const user$ = this.http.get<User>('/api/me');

// With shareReplay(1): only 1 request, result cached and shared
readonly user$ = this.http.get<User>('/api/me').pipe(
  shareReplay(1) // buffer 1 value, share with all subscribers
);

// All three get the same response from a single HTTP call:
this.user$.subscribe(u => this.header.setUser(u));
this.user$.subscribe(u => this.sidebar.setUser(u));
this.user$.subscribe(u => this.analytics.track(u));

Use in a service for cached data

cached-service.ts 
@Injectable({ providedIn: 'root' })
export class ReferenceDataService {
  // Fetched once, cached forever for all components
  readonly departments$: Observable<Department[]> =
    this.http.get<Department[]>('/api/departments').pipe(
      shareReplay(1)
    );

  readonly roles$: Observable<Role[]> =
    this.http.get<Role[]>('/api/roles').pipe(
      shareReplay(1)
    );
}
Angular Patterns

14. The async pipe

The async pipe subscribes to an Observable in the template and automatically unsubscribes when the component is destroyed. It's the cleanest way to use Observables in Angular.

users.component.ts 
@Component({
  template: `
    <div *ngIf="users$ | async as users; else loading">
      <div *ngFor="let user of users">{{ user.name }}</div>
    </div>

    <ng-template #loading>Loading...</ng-template>
  `
})
export class UsersComponent {
  // No subscribe(), no ngOnDestroy(), no Subscription variable
  users$ = this.http.get<User[]>('/api/users');
}

Combining multiple streams in template with combineLatest

dashboard.component.ts 
export class DashboardComponent {
  // Combine into a single stream to avoid multiple async pipes
  vm$ = combineLatest({
    user:   this.userService.user$,
    stats:  this.statsService.stats$,
    alerts: this.alertService.alerts$,
  });
}

// Template:
// <div *ngIf="vm$ | async as vm">
//   <h1>Welcome, {{ vm.user.name }}</h1>
//   <app-stats [data]="vm.stats"></app-stats>
//   <app-alerts [items]="vm.alerts"></app-alerts>
// </div>

15. Reactive forms + RxJS

valueChanges — react to form input

search.component.ts 
export class SearchComponent implements OnInit {
  searchControl = new FormControl('');
  results$!: Observable<SearchResult[]>;

  ngOnInit() {
    this.results$ = this.searchControl.valueChanges.pipe(
      debounceTime(400),
      distinctUntilChanged(),
      filter((q) => (q ?? '').length >= 2),
      switchMap((q) => this.searchService.search(q ?? '')),
      catchError(() => of([]))
    );
  }
}
// Use {{ results$ | async }} in template — zero subscriptions in .ts

statusChanges — react to form validity

form.component.ts 
ngOnInit() {
  // Enable submit button only when form becomes valid
  this.form.statusChanges.pipe(
    map((status) => status === 'VALID'),
    distinctUntilChanged(),
    takeUntilDestroyed(this.destroyRef)
  ).subscribe((valid) => this.submitEnabled = valid);

  // Cross-field dependency: show/hide a field based on another
  this.form.get('hasAddress')!.valueChanges.pipe(
    takeUntilDestroyed(this.destroyRef)
  ).subscribe((hasAddress) => {
    if (hasAddress) this.form.get('address')!.enable();
    else            this.form.get('address')!.disable();
  });
}

16. Router streams

employee-detail.component.ts 
export class EmployeeDetailComponent {
  // Stream that auto-loads employee whenever :id changes
  employee$ = this.route.paramMap.pipe(

  loading$ = this.router.events.pipe(
    map((e) => e instanceof NavigationStart),
  );
}

17. HttpClient & RxJS

HttpInterceptor for auth + error handling

auth.interceptor.ts 
@Injectable()
export class AuthInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<any>, next: HttpHandler) {
    const token = this.auth.getToken();
    const authReq = token
      ? req.clone({ setHeaders: { Authorization: `Bearer ${token}` } })
      : req;

    return next.handle(authReq).pipe(
      catchError((err: HttpErrorResponse) => {
        if (err.status === 401) this.auth.logout();
        if (err.status === 503) this.toast.show('Server unavailable');
        return throwError(() => err);
      })
    );
  }
}

Generic service with typed HTTP calls

employee.service.ts 
@Injectable({ providedIn: 'root' })
export class EmployeeService {
  private url = '/api/employees';

  getAll(): Observable<Employee[]> {
    return this.http.get<Employee[]>(this.url);
  }

  getById(id: number): Observable<Employee> {
    return this.http.get<Employee>(`${this.url}/${id}`);
  }

  save(emp: Employee): Observable<Employee> {
    return emp.id
      ? this.http.put<Employee>(`${this.url}/${emp.id}`, emp)
      : this.http.post<Employee>(this.url, emp);
  }

  delete(id: number): Observable<void> {
    return this.http.delete<void>(`${this.url}/${id}`);
  }
}

18. Reactive state service

A common pattern in Angular apps — a service that holds state in a BehaviorSubject and exposes derived streams. No NgRx needed for small-to-medium apps.

employee-list.service.ts 
interface EmployeeListState {
  employees: Employee[];
  loading:   boolean;
  error:     string | null;
  filter:    string;
}

@Injectable({ providedIn: 'root' })
export class EmployeeListService {
  private state$ = new BehaviorSubject<EmployeeListState>({
    employees: [],
    loading:   false,
    error:     null,
    filter:    '',
  });

  // Selectors — derived streams
  loading$   = this.state$.pipe(map((s) => s.loading), distinctUntilChanged());
  error$     = this.state$.pipe(map((s) => s.error),   distinctUntilChanged());
  employees$ = this.state$.pipe(
    map((s) => s.employees.filter(
      (e) => e.name.toLowerCase().includes(s.filter.toLowerCase())
    ))
  );

  // Actions
  load() {
    this.patch({ loading: true, error: null });
    this.http.get<Employee[]>('/api/employees').subscribe({
      next:  (employees) => this.patch({ employees, loading: false }),
      error: (err)       => this.patch({ error: err.message, loading: false }),
    });
  }

  setFilter(filter: string) { this.patch({ filter }); }

  private patch(partial: Partial<EmployeeListState>) {
    this.state$.next({ ...this.state$.value, ...partial });
  }
}

19. scan — accumulate state over time

scan works like Array.reduce but for streams — it builds up a running value over each emission.

scan.ts 
import { scan } from 'rxjs/operators';
import { Subject } from 'rxjs';

type Action = { type: 'ADD' | 'REMOVE' | 'CLEAR'; id?: number };

const action$ = new Subject<Action>();

const selectedIds$ = action$.pipe(
  scan((ids: Set<number>, action) => {
    const next = new Set(ids);
    if (action.type === 'ADD')    next.add(action.id!);
    if (action.type === 'REMOVE') next.delete(action.id!);
    if (action.type === 'CLEAR')  next.clear();
    return next;
  }, new Set<number>())
);

action$.next({ type: 'ADD', id: 1 });
action$.next({ type: 'ADD', id: 2 });
action$.next({ type: 'REMOVE', id: 1 });
// selectedIds$ emits: {1} → {1,2} → {2}

20. Memory leaks — the complete guide

Strategy When to use Requires ngOnDestroy?
async pipe Display data in template No
takeUntilDestroyed() Angular 16+, any subscription No
takeUntil(destroy$) Angular 15 and below Yes
take(1) / first() One-shot streams only No
HttpClient (no operator) Plain HTTP calls No — auto-completes
✅

Golden rule: If the Observable source never calls complete() (Subject, interval, WebSocket, valueChanges, router events), you must use one of the above strategies. When in doubt, use takeUntilDestroyed() — it never hurts.

clean-component.ts 
@Component({ ... })
export class CleanComponent implements OnInit {
  private destroyRef = inject(DestroyRef);

  // 1. async pipe — cleanest, zero boilerplate
  users$ = this.http.get<User[]>('/api/users');
  // template: {{ users$ | async }}

  ngOnInit() {
    // 2. takeUntilDestroyed for streams that affect non-template state
    interval(5000).pipe(
      switchMap(() => this.api.getStatus()),
      takeUntilDestroyed(this.destroyRef)
    ).subscribe((s) => this.status = s);

    // 3. take(1) for one-time initialization
    this.configService.config$.pipe(
      take(1)
    ).subscribe((cfg) => this.initWith(cfg));
  }
}

Quick Reference — Operator Cheat Sheet

  • Transform values: map, pluck, mapTo
  • Filter values: filter, distinctUntilChanged, debounceTime, throttleTime
  • Flatten inner streams: switchMap (cancel prev), mergeMap (parallel), concatMap (queue), exhaustMap (ignore while busy)
  • Combine streams: combineLatest (on any update), forkJoin (wait all), zip (paired), withLatestFrom (on one stream)
  • Limit stream: take, first, takeUntil, takeUntilDestroyed, skip
  • Error handling: catchError, retry, finalize
  • Multicast & cache: shareReplay, share
  • Side effects: tap (debug, setters), finalize (cleanup)
  • Accumulate state: scan, reduce
